# Target: CSS / Web Animations

Framework-free output that drops into any page. Prefer `@keyframes` +
`animation-*` for declarative sequences; use the Web Animations API
(`element.animate()`) when timing must be computed in JS.

## Spec → CSS
- `dur_ms` → `animation-duration` (e.g. `400ms`).
- `start_ms` (and `hold` before a move) → `animation-delay`.
- `ease` → `animation-timing-function` via `easing-map.md`
  (`ease-out`→`cubic-bezier(0.16,1,0.3,1)`, `linear`→`linear`, `spring`→`cubic-bezier(0.34,1.56,0.64,1)`).
- `props` `{p:[from,to]}` → the `0%` and `100%` keyframes. Combine transforms into one
  `transform:` (e.g. `translateY(24px) scale(0.9)`), animate `opacity` separately.
- `stagger.each_ms` → incremental `animation-delay` per child (`nth-child` or a
  `--i` custom property: `animation-delay: calc(var(--i) * 80ms)`).
- A trailing `hold` is automatic once the animation ends; set `animation-fill-mode: forwards`.

## Minimal runnable harness (`index.html`)

```html
<!doctype html>
<html>
<head><meta charset="utf-8"><style>
  body { margin:0; display:grid; place-items:center; height:100vh; background:#fff; }
  /* GENERATED BY motiscope — timing/easing measured; sizes/colors estimated. */
  .card {
    width:120px; height:120px; background:#3366ff; border-radius:12px;
    opacity:0;
    animation: reveal 400ms cubic-bezier(0.16,1,0.3,1) 0ms forwards;
  }
  @keyframes reveal {
    from { opacity:0; transform: translateY(24px) scale(0.9); }
    to   { opacity:1; transform: translateY(0)   scale(1);   }
  }
  @media (prefers-reduced-motion: reduce){
    .card { animation:none; opacity:1; transform:none; }
  }
</style></head>
<body><div class="card"></div></body>
</html>
```

## Web Animations API variant (when JS is preferred)
```js
card.animate(
  [ { opacity: 0, transform: "translateY(24px) scale(0.9)" },
    { opacity: 1, transform: "translateY(0) scale(1)" } ],
  { duration: 400, delay: 0, easing: "cubic-bezier(0.16,1,0.3,1)", fill: "forwards" }
);
```
Respect reduced motion by checking `matchMedia("(prefers-reduced-motion: reduce)")`
and jumping to the end state instead of animating.
