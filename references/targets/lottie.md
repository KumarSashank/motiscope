# Target: Lottie / SVG

Be honest about scope. True Lottie (bodymovin) files are authored in After Effects
with full vector data that **cannot be recovered from frames**. From a screen
recording, prefer one of these, in order:

1. **Animated SVG (CSS or SMIL)** — best when the animation is transform/opacity of
   simple vector shapes. This is the recommended default for "Lottie-style" motion.
2. **A minimal, clearly-labeled-approximate bodymovin JSON** — only for simple
   transform-only shape layers (position/scale/rotation/opacity of a rect/ellipse).
   Mark it approximate and note that production Lottie should be authored in AE.

Always tell the user which one you produced and why.

## Animated SVG (SMIL) template
```html
<!-- GENERATED BY motiscope — timing/easing measured; shapes/colors estimated.
     For production-grade Lottie, author in After Effects + bodymovin. -->
<svg xmlns="http://www.w3.org/2000/svg" width="240" height="240" viewBox="0 0 240 240">
  <rect x="60" y="60" width="120" height="120" rx="12" fill="#3366ff" opacity="0">
    <animate attributeName="opacity" from="0" to="1" dur="0.4s"
             begin="0s" fill="freeze"
             keyTimes="0;1" keySplines="0.16 1 0.3 1" calcMode="spline"/>
    <animateTransform attributeName="transform" type="translate"
             from="0 24" to="0 0" dur="0.4s" begin="0s" fill="freeze"
             keyTimes="0;1" keySplines="0.16 1 0.3 1" calcMode="spline"/>
  </rect>
</svg>
```
Map `ease` → `keySplines` using the CSS cubic-bezier control points from
`easing-map.md`; `dur_ms`→`dur`; `start_ms`/`hold`→`begin`.

## Minimal bodymovin JSON (transform-only, approximate)
Only if the user specifically wants a `.json` for `lottie-web`:
- `ip`/`op` = in/out point in frames (`fr` = frame rate, e.g. 60).
- One shape layer per element; animate `ks.p` (position), `ks.s` (scale), `ks.r`
  (rotation), `ks.o` (opacity) with two keyframes each and bezier `i`/`o` handles
  derived from the ease.
- State clearly in a comment that shapes/colors are estimated from pixels and this is
  not a substitute for AE-authored Lottie.

Prefer the SVG route unless a `.json` is explicitly required.
