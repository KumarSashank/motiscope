# Target: Framer Motion (React)

`motion.*` components for React / Next.js. Install: `npm i framer-motion`.

## Spec → Framer Motion
- `props` `{p:[from,to]}` → `initial` (the `from`) and `animate` (the `to`).
- `dur_ms` → `transition.duration` (seconds); `start_ms`/`hold` → `transition.delay`.
- `ease` → `transition.ease` via `easing-map.md` (`ease-out`→`"easeOut"`, `linear`→`"linear"`);
  for `spring` use `transition:{ type:"spring", stiffness:300, damping:20 }` and tell the
  user to tune stiffness/damping (physics can't be recovered from frames).
- `stagger` → parent `variants` with `transition:{ staggerChildren: each_ms/1000 }`,
  or `custom` + a delay function; `from:"start"|"center"|"end"` picks the order.
- `y`/`x` map to the transform shorthands Framer accepts directly.

## Component template

```tsx
// GENERATED BY motiscope — timing/easing measured; sizes/colors estimated.
import { motion, useReducedMotion } from "framer-motion";

export function Reveal() {
  const reduce = useReducedMotion();
  return (
    <motion.div
      initial={reduce ? false : { opacity: 0, y: 24, scale: 0.9 }}
      animate={{ opacity: 1, y: 0, scale: 1 }}
      transition={{ duration: 0.4, delay: 0, ease: "easeOut" }}
      style={{ width: 120, height: 120, background: "#3366ff", borderRadius: 12 }}
    />
  );
}
```

## Staggered list template
```tsx
const container = { animate: { transition: { staggerChildren: 0.08 } } };
const item = {
  initial: { opacity: 0, y: 24 },
  animate: { opacity: 1, y: 0, transition: { duration: 0.4, ease: "easeOut" } },
};
// <motion.ul variants={container} initial="initial" animate="animate">
//   {items.map(i => <motion.li key={i} variants={item} />)}
```
`useReducedMotion()` → skip the `initial` offset so the element renders in its final state.
